# Датасет сведений о ярчайших звёздах неба до 6,5m

Датасет в формате CSV сгенерирован на основе астрономических каталогов Hipparcos, Pastel и др. и содержит 8874 записи.  Также опубликован скрипт, генерирующий файлы данных из исходных каталогов. Примеры работы с данными на Python/Pandas и R. Предназначен для учебных целей и любительской астрономии.

## Файлы для скачивания

**dataset_bright_stars.csv** - датасет в формате стандартного CSV (разделители - запятые, кодировка “Юникод UTF-8”).

Может быть прочитан скриптом на любом языке программирования, в том числе на Python/Pandas и R, или загружен в базу данных, такие, как [DuckDB](https://duckdb.org/). Его можно открыть и в электронной таблице.

{% note info "Региональные установки для чтения CSV" %}

Разделители колонок - запятые, дробная часть отделяется точкой - это соответствует локали “Английский (США)”. [Libre Office Calc](https://ru.libreoffice.org/) при открытии файла запрашивает эти параметры и, даже, определяет автоматически. Более того, в настройках самого приложения можно изменить только для него региональные установки. Тогда и в приложении дробная часть будут отделяться точкой, как в языках программирования.\
В отличие от Calc, Excel воспринимает региональные установки операционной системы.

{% endnote %}

**dataset_bright_stars_excel.csv** - датасет для тех, кто не хочет отказываться от Excel. \
Разделители - точка с запятой, дробная часть отделается запятой. Кодировка - стандартная для Windows cp1251.

В варианте Hipparcos, распространяемых на CD (что также можно скачать) есть папка `tables`. В файле `ident4.doc` приведены обозначения звёзд по Байеру и Флемстиду, а в `ident5.doc` - обозначения переменных звёзд. Последние вошли в датасет в поле `VarID`. А для байеровских и флемстидовских обозначений использован астрономический каталог [“HD-DM-GC-HR-HIP-Bayer-Flamsteed Cross Index” (Kostjuk, 2002)](https://cdsarc.cds.unistra.fr/viz-bin/cat/IV/27A). В этих источниках греческие буквы обозначены строго тремя латинскими буквами (когда используется только две буквы, то добавляется точка).

В обозначениях по Байеру и Флемстиду фигурирует название созвездия. Иногда звезда по факту находится на территории другого созвездия из-за того, что до 1924 г. не были точно установлены границы созвездий. Также из-за собственного движения звезда может перейти в другое созвездие. Границы созвездия определены в файлах на [странице Международного астрономического союза](https://iau.org/public/themes/constellations/). Там же приведён список сокращённых обозначений созвездий.

Был написан скрипт [constellations.py](http://constellations.py), который читает эти файлы и определяет созвездие для каждой звезды. Поэтому поле Cst содержит именно расчётную, актуальную принадлежность к созвездию. Иногда она не совпадает с байеровским, поэтому этой информации отведено отдельное поле. Благодаря такому полю и охвату 100 % можно вывести звёзды, принадлежащие одному или нескольким определённым созвездиям.

Собственные имена звёзд. Они есть и в файле `ident6.doc` из комплекта Hipparcos, и в файле `table3.dat` каталога от Kostjuk. Также есть [таблица от Международного астрономического союза](https://www.iau.org/public/themes/naming_stars/). Но в них названия на английском (в последнем источнике - на языке оригинала). В результате русские и английские имена были взяты из [таблицы в Википедии](https://ru.wikipedia.org/wiki/%D0%A1%D0%BF%D0%B8%D1%81%D0%BE%D0%BA_%D0%BD%D0%B0%D0%B7%D0%B2%D0%B0%D0%BD%D0%B8%D0%B9_%D0%B7%D0%B2%D1%91%D0%B7%D0%B4), убраны странные имена звёзд, добавлены недостающие номера HIP и записаны в файл star_names_wiki_rus.csv .

{% note info "Как искать астрономические каталоги под свои цели" %}

В целом, искать подходящие каталоги можно не только через поисковую машину. Есть, например, FTP-сервер **[cdsarc.u-strasbg.fr](http://cdsarc.u-strasbg.fr)**, где cобраны многие из них. Следует найти файл со списком каталогов (`cats-list.txt` или `cats.lis`) , в котором приведён список размещённых каталогов с кратким описанием. В настоящее время во многих браузерах доступ по незащищённому FTP запрещён, поэтому удобно использовать приложение наподобие [Filezilla](https://filezilla-project.org/) (Client). Для анонимного доступа логин: anonymous, пароль: любые буквы со значком @ на конце.

{% endnote %}

## Двойные и кратные звёзды

Многие каталоги содержат записи, за которым кроется два или более звёздных компонентов. В Hipparcos есть поле Ncomp, в котором написано, сколько этих компонентов, а в поле m_HIP показаны их буквенные обозначения. С точки зрения астрофизики все компоненты - это отдельные звёзды, к ним можно было бы давать отдельные характеристики (например, эффективную температуру). С точки зрения наблюдательной астрономии невооружённый глаз редко видит два компонента, обычно это одна звезда.

В данном датасете решено оставить так, как уже решили в Hipparcos. Предстоит ещё решить проблему, когда в системе два компонента (или две группы компонентов) обозначены разыми номерами HIP. Например, систма α Cen - звёзды Толиман HIP 71681 и Ригиль Кентаврус HIP 71683.

Приведены в каталоге `src`. Также потребуется скачать следующие астрономические каталоги:

- Hipparcos, файл `hip_main.dat` нужно разместить в папку hipparcos ( **[cdsarc.u-strasbg.fr/0/cats/I/239](http://cdsarc.u-strasbg.fr/0/cats/I/239)** )
